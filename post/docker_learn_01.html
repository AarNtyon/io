docker 极简部署

Rain, Sat Jun 10 2023back

第一次接触docker的时候或多或少有点奇怪，docker的功能是什么，怎么用，方便吗

接下里，完成一个极简的docker部署

新建一个文件夹，在文件夹里面新增一个index.js文件

console.log('hello docker!')

此时我们在本地当前目录中运行 node index.js ,我们可以认为这个项目就已经跑起来了

但是，如果我们想把这一个文件运行在线上服务器中，我们需要在服务器上下载安装node运行时，接着再进入到目录中再次执行启动脚本，才算是把服务器启动起来。而这个过程我们会发现是有点割裂的，我们上传了index.js 文件，还得再上去服务器上安装一遍node运行时。而小项目没什么环境依赖还算简单，但如果你的项目有mysql，redis，mongodb呢，在本地开发的时候你需要把这些环境还有依赖安装一遍，等你上线的时候，你还得再来一遍。

而docker可以很好的帮我们解决这个问题，你可以把docker想象成一个黑盒，把你需要用到的环境依赖全部扔进去，你不用管内部做了什么操作，只需要执行几行docker命令，就可以启动你的项目，同样的，等你上线的时候，你也只是做一样的事情，因为影响你正常启动项目的环境或是依赖操作，docker都帮你解决了

# 选择镜像
FROM node:latest

#设置工作目录 代表以后的RUN/CMD命令都在这个工作目录中运行
WORKDIR /runDir

# 把当前的代码都添加到这个努力中
ADD . /runDir

# 安装依赖
RUN yarn

# 启动项目
CMD node index.js
